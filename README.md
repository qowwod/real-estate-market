## SQL-проект: Анализ данных недвижимости Санкт-Петербурга и Ленинградской области
## Описание проекта

Проект посвящён анализу объявлений о продаже квартир в Санкт-Петербурге и населённых пунктах Ленинградской области.
Данные исходно представлены в базе data-analyst-estate (Яндекс Недвижимость), однако из-за конфиденциальности реальные данные не публикуются. В репозитории представлена структура базы данных и фиктивные примеры, чтобы проект можно было воспроизвести и протестировать.

### Структура базы данных
- Таблица `advertisement` содержит информацию об объявлениях о продаже недвижимости.
| Поле                 | Тип      | Описание                                                       |
| -------------------- | -------- | -------------------------------------------------------------- |
| id                   | INT (PK) | Идентификатор объявления                                       |
| first_day_exposition | DATE     | Дата подачи объявления                                         |
| days_exposition      | INT      | Длительность публикации (дней), NULL — если объявление активно |
| last_price           | DECIMAL  | Стоимость квартиры (руб.)                                      |


- Таблица `flats` cодержит характеристики квартир.
| Поле             | Тип      | Описание                                          |
| ---------------- | -------- | ------------------------------------------------- |
| id               | INT (PK) | Идентификатор квартиры, связан с advertisement.id |
| city_id          | INT (FK) | Ссылка на город (city.city_id)                    |
| type_id          | INT (FK) | Ссылка на тип населённого пункта (type.type_id)   |
| total_area       | DECIMAL  | Общая площадь (м²)                                |
| rooms            | INT      | Количество комнат                                 |
| ceiling_height   | DECIMAL  | Высота потолка (м)                                |
| floors_total     | INT      | Этажность дома                                    |
| living_area      | DECIMAL  | Жилая площадь (м²)                                |
| floor            | INT      | Этаж квартиры                                     |
| is_apartment     | BOOLEAN  | 1 - апартаменты, 0 - нет                          |
| open_plan        | BOOLEAN  | 1 - открытая планировка, 0 - нет                  |
| kitchen_area     | DECIMAL  | Площадь кухни (м²)                                |
| balcony          | INT      | Количество балконов                               |
| airports_nearest | INT      | Расстояние до ближайшего аэропорта (м)            |
| parks_around3000 | INT      | Количество парков в радиусе 3 км                  |
| ponds_around3000 | INT      | Количество водоёмов в радиусе 3 км                |

- Таблица `city` cодержит информацию о городах и населённых пунктах.
| Поле    | Тип      | Описание             |
| ------- | -------- | -------------------- |
| city_id | INT (PK) | Идентификатор города |
| city    | VARCHAR  | Название города      |

- Таблица `type` cодержит типы населённых пунктов.
| Поле    | Тип      | Описание                              |
| ------- | -------- | ------------------------------------- |
| type_id | INT (PK) | Идентификатор типа населённого пункта |
| type    | VARCHAR  | Название типа населённого пункта      |

## Зависимости между таблицами
advertisement.id  →  flats.id
flats.city_id     →  city.city_id
flats.type_id     →  type.type_id

| days_exposition      | INT      | Длительность публикации (дней), NULL — если объявление активно |
| last_price           | DECIMAL  | Стоимость квартиры (руб.)                                      |
